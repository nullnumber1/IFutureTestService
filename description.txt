Краткое описание проекта и принятых решений:
– В качестве стека для создания сервиса был выбран Spring Boot + Spring Data JPA для работы с БД
– На уровне Entity и репозиторного класса прописана аннотация для кэширования данных. Кэширование корректно работает на селектах, было проверено с помощью включения логгинга гибернейта. После операции обновления таблицы, кэш для конкретной сущности очищается по понятным причинам.
– Для мониторинга был создан MBean посредством аннотаций. Через JConsole можно посмотреть актуальное значение аттрибутов counterTotal и counterPerHour, а также вызвать метод resetCounters. Так же создан ScheduledTask, который каждый час обнуляет значение counterPerHour.
– Так же для операций чтения и обновления были прописаны транзакции с уровнем изоляции Serializable. Это сделано для обеспечения целостности прочитанных данных в случае параллельного обновления таблицы.
– Для возможности одновременного запуска нескольких клиентов, в application.properties обозначен порт приложения, который можно менять при сборке очередной реплики. Поскольку я спешил, он ,можно считать, захардкожен. Если делать по-человечески, то я бы сделал docker-compose конфигурацию, где порт при запуске брался из .env файла и передавался бы подстановкной переменной в application.properties.
